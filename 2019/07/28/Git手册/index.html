网上大多介绍云里雾里，这里谈下个人理解。

git前面知道是分布式版本控制系统，何谓分布式，在git每次你所进行提交的内容都会有个唯一的id，称之为版本号(commit id)，而这个id就是每个版本的内容修改记录，类似于时间与空间的对应性，每个时间都有一个空间，而git每次进行回退都是在使用这个id进行选择你的文件内容，在这个操作中类似于时空穿梭。而git add和git commit的操作就是根据修改形成一个版本号，也就是git的一个分支，

总结

  • 回到git add或git commit的状态，工作区修改未提交的全部丢弃。
  • 修改了工作区内容，添加到了暂存区，丢弃修改

  • 修改了工作区内容，添加到了暂存区，并且提交了分支，丢弃修改。

    如上，只能版本回退。
